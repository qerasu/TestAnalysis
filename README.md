Workflow для автоматической генерации отчета

После запуска автоматически формируется интерактивный HTML-дашборд `output/report.html`:

- KPI-карточки (выручка, продажи, возвраты, топ-товар, кол-во алертов)
- 6 интерактивных графиков (динамика выручки, категории, товары, возвраты, остатки, причины возвратов)
- Таблица топ-5 товаров по выручке
- Таблица проблемных позиций с тегами
- Блок алертов с приоритетами (критические / предупреждения / инфо)
- Автоматически сгенерированное текстовое резюме

---

## Как организован процесс

```
data/ (CSV + TXT)
    │
loader.py → загрузка и валидация входных файлов
    │
metrics.py → расчёт всех KPI
    │
alerts.py → генерация алертов
    │
summary.py → формирование текстового резюме
    │
report.py → сборка HTML-дашборда
    │
output/report.html → открывается в браузере
```

Весь цикл запускается одной командой:

```bash
python run.py
```

---

## Что автоматизировано

| Шаг | Что происходит |
|---|---|
| Загрузка данных | чтение CSV, нормализация типов, валидация колонок |
| Расчёт KPI | выручка, продажи, возвраты, топы, аномалии, конверсия |
| Алерты | автоматическое выделение проблемных позиций по 5 правилам |
| Резюме | формирование текстового вывода на основе данных |
| Отчёт | генерация HTML с графиками, таблицами и алертами |
| Открытие | отчёт автоматически открывается в браузере |

---

## Как обновляются данные

При замене входных файлов (`sales.csv`, `returns.csv`, `stock.csv`, `campaign_info.txt`) достаточно заново запустить цикл:

```bash
python run.py
```

Все показатели, алерты, summary и проблемные позиции пересчитываются и перезаписываются в `output/report.html` автоматически.

## Рассчитываемые показатели

- **Общая выручка** — сумма по всем SKU за период
- **Количество продаж** — суммарное кол-во проданных единиц
- **Общий % возвратов** — возвраты / продажи
- **Топ-5 товаров** — по суммарной выручке
- **Проблемные товары** — высокий % возвратов, низкие продажи, низкий остаток
- **Аномальные показатели** — Z-score выручки > 2.0
- **Динамика выручки** — по дням периода
- **Структура по категориям** — доли выручки
- **Причины возвратов** — топ по количеству

---

## Логика алертов

| Тип | Условие | Приоритет |
|---|---|---|
| Высокий % возвратов | > 20% | 🔴 Критический |
| Критически низкий остаток | < 20 шт. | 🔴 Критический |
| Повышенный % возвратов | 15–20% | 🟡 Предупреждение |
| Низкий остаток | 20–50 шт. | 🟡 Предупреждение |
| Дни без продаж при наличии остатка | ≥ 3 дней | 🟠 Предупреждение |
| Низкие продажи | нижние 15% по выручке | 🔵 Инфо |
| Аномальная выручка | Z-score > 2.0 | ⚠️ Инфо |

Пороги можно настроить в `config.py`.

## Запуск

### Активировать виртуальное окружение

```bash
# для macOS / Linux:
source .venv/bin/activate
```

```cmd
# для Windows
.venv\Scripts\activate
```

## Установить зависимости
```bash
pip install -r requirements.txt

# запустить workflow
python run.py
```

Отчёт откроется в браузере автоматически. Путь к файлу: `output/report.html`.
